# 飞书多维表格集成配置指南

## 一、创建飞书自建应用

### 1. 访问飞书开放平台
- 打开 https://open.feishu.cn/
- 使用企业管理员账号登录

### 2. 创建企业自建应用
1. 点击"创建企业自建应用"
2. 填写应用名称：**车辆预定系统**
3. 上传应用图标（可选）
4. 点击"创建"

### 3. 获取应用凭证
创建完成后，在应用详情页面可以看到：
- **App ID**（记下来）
- **App Secret**（点击查看并记下来）

### 4. 配置应用权限
在"权限管理"页面，添加以下权限：
- `bitable:app` - 查看、编辑和管理多维表格
- `bitable:app:readonly` - 获取多维表格元数据

点击"发布版本"使权限生效。

## 二、创建飞书多维表格

### 1. 创建多维表格文档
1. 在飞书工作台点击"创建" > "多维表格"
2. 命名为：**车辆预定系统数据库**

### 2. 创建"车辆信息"数据表

点击创建新表，命名为"车辆信息"，添加以下字段：

| 字段名称 | 字段类型 | 说明 | 是否必填 |
|---------|---------|------|---------|
| 车辆ID | 文本 | 主键，如VEH001 | 是 |
| 车辆编码 | 文本 | 资产编码，如765772 | 是 |
| 车辆型号 | 文本 | 如Model3 | 是 |
| 车系 | 文本 | 如特斯拉 | 是 |
| 车辆 | 文本 | 如Model3 | 是 |
| 车辆位置 | 文本 | 如北京市 | 否 |
| 车辆城市 | 文本 | 如北京市 | 是 |
| VIN码 | 文本 | 车架号 | 是 |
| 车辆阶段 | 单选 | SOP/PPV/EP2/P等 | 是 |
| 内外饰颜色 | 文本 | 车辆颜色描述 | 否 |
| 车牌号 | 文本 | 如京ACV6773 | 否 |
| 车辆图片 | 文本 | 图片URL | 否 |

### 3. 创建"预定记录"数据表

创建第二个表，命名为"预定记录"，添加以下字段：

| 字段名称 | 字段类型 | 说明 | 是否必填 |
|---------|---------|------|---------|
| 预定ID | 文本 | 自动生成，如BOOK1234567890 | 是 |
| 车辆 | 关联 | 关联到"车辆信息"表 | 是 |
| 车辆ID | 文本 | 如VEH001 | 是 |
| 车辆型号 | 文本 | 如Model3 | 是 |
| 申请原因 | 文本 | 多行文本 | 是 |
| 开始时间 | 日期时间 | 使用开始时间 | 是 |
| 结束时间 | 日期时间 | 使用结束时间 | 是 |
| 申请人 | 文本 | 姓名 | 是 |
| 创建时间 | 日期时间 | 预定提交时间 | 是 |
| 状态 | 单选 | 待审批/已批准/已拒绝/已还车 | 是 |
| 是否已还车 | 复选框 | 布尔值 | 否 |
| 还车时间 | 日期时间 | 实际还车时间 | 否 |

### 4. 导入初始车辆数据

在"车辆信息"表中，可以：
- 手动添加车辆记录，或
- 从CSV文件导入（使用之前的产品部车辆管理清单.csv）

导入步骤：
1. 点击表格右上角"更多" > "导入数据"
2. 选择CSV文件
3. 映射字段对应关系
4. 确认导入

### 5. 获取多维表格配置信息

**获取 App Token：**
1. 打开多维表格
2. 点击右上角"..."菜单 > "高级设置"
3. 复制 **App Token**（一串以 `bascn` 开头的字符串）

**获取 Table ID：**
1. 点击"车辆信息"表标签
2. 查看浏览器地址栏，URL格式为：
   ```
   https://xxx.feishu.cn/base/bascnXXXXXXX?table=tblXXXXXXXX
   ```
3. `table=` 后面的就是 **Table ID**（以 `tbl` 开头）
4. 同样方法获取"预定记录"表的 Table ID

## 三、配置服务器环境变量

### 1. 创建 .env 文件
在 `server` 文件夹中，复制 `.env.example` 为 `.env`：

```bash
cp .env.example .env
```

### 2. 填写配置信息
编辑 `.env` 文件，填入您在前面步骤获取的信息：

```env
# 飞书应用配置
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# 飞书多维表格配置
FEISHU_APP_TOKEN=bascnXXXXXXXXXXXXXXXXXXX
FEISHU_VEHICLES_TABLE_ID=tblXXXXXXXXXXXXXXXX
FEISHU_BOOKINGS_TABLE_ID=tblYYYYYYYYYYYYYYYY

# 服务器配置
PORT=3000
```

## 四、安装和启动服务器

### 1. 安装依赖
打开命令行，进入 `server` 文件夹：

```bash
cd C:\Users\gaoshujun\vehicle-booking-system\server

```

### 2. 启动服务器
```bash

```

如果看到以下输出，说明服务器启动成功：
```

📋 健康检查: http://localhost:3000/api/health
```

在浏览器访问：http://localhost:3000/api/health

如果返回 `{"status":"ok","message":"服务运行正常"}`，说明配置成功。

## 五、部署到公网（必需）

飞书应用需要通过HTTPS访问，所以需要部署到公网。推荐以下免费方案：

### 方案A：Render（推荐）

1. 访问 https://render.com
2. 注册账号
3. 点击 "New +" > "Web Service"
4. 连接GitHub仓库或直接上传代码
5. 配置：
   - **Build Command**: `cd server && npm install`
   - **Start Command**: `cd server && npm start`
6. 添加环境变量（将.env中的配置复制过去）
7. 点击"Create Web Service"
8. 等待部署完成，获得公网URL（如 https://xxx.onrender.com）

### 方案B：Railway

1. 访问 https://railway.app
2. 注册账号
3. 点击 "New Project" > "Deploy from GitHub"
4. 选择仓库
5. 添加环境变量
6. 自动部署

## 六、配置飞书自建应用

### 1. 配置网页地址
回到飞书开放平台，在应用管理页面：
1. 点击"应用功能" > "网页"
2. 配置：
   - **桌面端首页**: `https://你的域名/index.html`
   - **移动端首页**: `https://你的域名/index.html`
3. 保存配置

### 2. 配置回调地址（如需）
如果需要用户身份验证：
1. 在"安全设置"中配置重定向URL
2. 添加：`https://你的域名/callback`

### 3. 发布应用
1. 点击"版本管理与发布"
2. 创建版本
3. 提交审核（企业内部应用审核很快）
4. 审核通过后发布

## 八、常见问题

### Q1: API调用失败，返回 "无权限"
**解决方法：**
- 检查应用权限是否已添加并发布
- 确认App ID和App Secret是否正确
- 尝试重新获取tenant_access_token

### Q2: 无法写入多维表格
**解决方法：**
- 检查App Token和Table ID是否正确
- 确认字段名称与代码中的映射一致
- 查看飞书开放平台的API文档确认字段类型

### Q3: 本地测试正常，部署后无法访问
**解决方法：**
- 确认环境变量已正确配置到部署平台
- 检查部署平台的日志
- 确认使用HTTPS而不是HTTP

### Q4: 多人同时预定时产生冲突
**解决方法：**
- 在服务器端添加锁机制
- 使用飞书多维表格的自动化功能检测冲突
- 添加预定审批流程
